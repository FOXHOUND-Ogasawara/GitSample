今のプリリパさんの作りは、HighAndLowゲームを続けるたびに毎回カードを引いてそれに対するジャッジになっています（1回目の挙動が繰り返される）。

■コード
・無駄な記述がある（import、変数宣言）
・ユーザーネームの代入でuserName = userNameは不要
→do～whileの中身を
do {
      System.out.println("Does not match condition of the username");
      userName = GameUtils.getInputString();// 入力習得
} while (!GameUtils.checkPattern(userName));// 繰り返し案件
で問題ないです。
・また、あまり実務でdo～whileを使うことは少ないです。whileの方が一般的ですので、
while (!GameUtils.checkPattern(userName)) {
　　System.out.println("Does not match condition of the username");
　　userName = GameUtils.getInputString();// 入力習得
}
でOKです。
do～whileを使わない理由としては、ループ条件が下にあると読みづらいためです（中身の処理が少なければいいですが、コード量が増えた時に読みづらい）
他のところでもdo~whileを使用していますが、基本的にはwhileで良いです。
